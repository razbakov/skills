# Decision-Making Patterns

These patterns supplement the consent decision-making covered in the governance phase. They apply whenever agents or humans need to make, challenge, or evolve decisions.

## Resolve Objections

When an objection is raised (by an agent in a PR, or by a human in review), follow these steps to resolve it:

1. **Take one objection at a time.** Don't try to address multiple objections simultaneously.
2. **Come up with a proposal for an amendment:**
   - First, ask the person who raised the objection if they have a suggestion.
   - If not, ask the wider group for suggestions.
   - If still unclear, invite a time-boxed dialogue to generate ideas.
3. **Understand the amendment** — clarifying questions only, no evaluation yet.
4. **Check for objections to the amendment** — does anyone see a problem with the proposed fix?
5. **If new objections arise**, resolve them recursively (amendment to the amendment).
6. **Once resolved**, integrate the amendment into the original proposal and move to the next objection.

A proposal only becomes a policy/decision once ALL objections have been resolved.

**Options when resolving is difficult:**
- Add, remove, or change something in the proposal
- Defer resolution to a specific person, with a deadline
- Create an entirely new proposal
- Delegate research to an individual or group
- Leave the proposal unchanged and monitor the outcome
- Ask the delegator for input
- Take more time for reflection and revisit later

**For AI agent organizations:** When an agent raises an objection in a PR description (in the Tensions section), the delegator must resolve it before merging — either by amending the work, asking the agent to revise, or documenting why the objection is noted but not blocking.

## Co-Create Proposals

When a decision requires input from multiple roles or agents, use this collaborative process:

1. **Agree on the driver and requirement** — what situation are we responding to and what do we need?
2. **Explore the topic** — gather constraints, resources, context from all affected parties.
3. **Generate ideas** — each agent/person contributes possible approaches.
4. **Design a proposal** — delegate drafting to 1-2 people (tuners), not the whole group.

The key insight: **separate ideation from drafting.** The group contributes ideas; a small team synthesizes them into a coherent proposal. Don't draft by committee.

**For AI agent organizations:** When a decision affects multiple agents (e.g., data schema used by both Engineer and Operations Manager):
1. Coordinator identifies the affected agents
2. Each agent is dispatched to contribute their perspective (constraints, needs, ideas)
3. One agent is designated as "tuner" to draft the proposal
4. The proposal goes through consent (delegator reviews, other agents can raise objections)

## Proposal Forming (Structured Variant)

A more structured version of Co-Create Proposals with 7 steps. Use when the topic is complex or when agents have conflicting needs:

1. **Consent to purpose** — does everyone agree this is worth addressing?
2. **Questions about the purpose** — deepen understanding
3. **Information gathering** — document constraints, resources, open questions
4. **Generative questions** — what requirements must the proposal fulfill? Sort into Essential / Desirable / Maybe Later
5. **Collect ideas** — everyone contributes possible solutions
6. **Choose tuners** — select 1-2 agents/people to draft the proposal
7. **Tuners design proposal** — they synthesize everything into a draft for group review

## Reasoned Decision-Making

The foundational process for reaching agreement using reason (used inside consent, objection testing, and objection resolution):

1. **Present** the subject (argument, proposal, or question)
2. **Understand** — clarifying questions only
3. **Check** — does anyone disagree? (If no, skip to 6)
4. **Dispute** — hear the reasoning behind disagreement
5. **Integrate** — improve the subject based on what was learned, then go back to 3
6. **Celebrate** — agreement reached

## Role Selection

When assigning a person or agent to a role, prefer consent-based selection over simple assignment:

1. **Present the role description** (Delegation Canvas)
2. **Nominations** — each participant nominates someone (or themselves) with a reason
3. **Share reasons** — why did you nominate this person?
4. **Information gathering** — what else should we know?
5. **Nomination changes** — anyone want to change their nomination based on what was shared?
6. **Propose a nominee** — facilitator identifies the candidate with the strongest reasons
7. **Check for objections** — including from the nominee themselves
8. **Resolve objections** if any
9. **Celebrate** the selection

**For AI agent organizations:** Role selection is less relevant since AI agents don't have personal preferences. However, when choosing which human is the best delegator for a role, or when deciding whether to split/merge agent roles, this process ensures the decision is well-reasoned.
